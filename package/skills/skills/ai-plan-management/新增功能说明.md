# AI计划管理系统 OpenClaw Plugin - 新增功能说明

本文档说明新增的两个核心功能:**修改计划**和**计划提醒**。

## 🔧 功能1: 修改计划内容

### 使用场景

| 场景 | 用户说法 | 效果 |
|------|---------|------|
| 延长deadline | "把减肥计划的截止日期延长到4月1日" | 修改 deadline 字段 |
| 修改重要性 | "把工作计划的重要性改成8" | 修改 importance 字段 |
| 更改分类 | "把这个计划改成学习类" | 修改 category 字段 |
| 标记完成 | "我已经完成了减肥计划" | 修改 status 为 completed |

### 工具: `update_plan`

**参数**:
```typescript
{
  plan_id: string;           // 必需,从上下文或查询获取
  title?: string;            // 可选,新的计划名称
  deadline?: string;         // 可选,YYYY-MM-DD
  deadlineTime?: string;     // 可选,HH:mm
  startDate?: string;        // 可选,YYYY-MM-DD
  startTime?: string;        // 可选,HH:mm
  category?: string;         // 可选,工作/学习/生活等
  importance?: number;       // 可选,1-10
  urgency?: number;          // 可选,1-10
  executionNotes?: string;   // 可选,备注
  status?: string;           // 可选,pending/in_progress/completed/blocked
}
```

### 完整对话示例

```
用户: "我有哪些计划?"

AI: 📋 您当前有 3 个计划:

    1. 30天减肥计划 (运动) ⏳
       截止: 2026-03-29

    2. 学习Python (学习) 🔄

    3. 准备年度报告 (工作) ✅

    [保存到上下文: recent_plans]

─────────────────────────────

用户: "把第1个计划的截止日期延长到4月1日"

AI: [从 recent_plans 提取: index=1 → plan_id="plan_abc"]
    [识别: 字段=deadline, 新值="2026-04-01"]

    您要将"30天减肥计划"的截止日期从 2026-03-29 改为 2026-04-01,是否确认?

用户: "确认"

AI: [调用 update_plan(plan_id="plan_abc", deadline="2026-04-01")]

    ✅ 计划已成功更新!

    📋 计划ID: plan_abc
    📝 更新的字段: deadline, dueDate

    📊 变更详情:
      • deadline: 2026-03-29 → 2026-04-01
      • dueDate: 2026-03-29 → 2026-04-01
```

### SKILL处理规则

1. **智能引用计划**:
   - "第X个计划" → 从 `recent_plans` 提取
   - "减肥计划" → 模糊匹配标题
   - 直接提供 plan_id

2. **智能识别字段**:
   - "截止日期" / "deadline" → `deadline`
   - "重要性" / "importance" → `importance`
   - "分类" / "类别" → `category`

3. **智能提取新值**:
   - 日期: "4月1日" → "2026-04-01"
   - 数字: "8" → 8
   - 状态: "完成" → "completed"

4. **安全确认**:
   - 所有修改操作前复述变更
   - 等待用户确认
   - 可以取消

## ⏰ 功能2: 计划提醒

### 使用场景

| 场景 | 用户说法 | 效果 |
|------|---------|------|
| 主动查询 | "有什么需要提醒的吗?" | 返回所有提醒 |
| 逾期检查 | "我有逾期的计划吗?" | 返回逾期计划 |
| 到期检查 | "有哪些计划要到期了?" | 返回即将到期的计划 |

### 工具: `get_plan_reminders`

**无参数** - 自动检查用户的所有计划

**返回数据**:
```typescript
{
  total_reminders: number;
  reminders: {
    overdue: Array<{          // 已逾期(优先级最高)
      plan_id: string;
      title: string;
      category: string;
      deadline: string;
      days_overdue: number;
      reason: string;
    }>;
    blocked: Array<{          // 被阻塞
      plan_id: string;
      title: string;
      status: "blocked";
      reason: string;
    }>;
    due_soon: Array<{         // 即将到期(3天内)
      plan_id: string;
      title: string;
      deadline: string;
      days_left: number;
      reason: string;
    }>;
    stale: Array<{            // 长时间未更新(7天+)
      plan_id: string;
      title: string;
      days_stale: number;
      reason: string;
    }>;
  };
  summary: {
    overdue_count: number;
    due_soon_count: number;
    stale_count: number;
    blocked_count: number;
  };
}
```

### 完整对话示例

```
用户: "有什么需要提醒的吗?"

AI: [调用 get_plan_reminders 工具]

    ⏰ 计划提醒 (共 4 项需要关注)

    🔴 已逾期的计划:
      1. 准备年度报告 (工作)
         已逾期 2 天
         截止: 2026-03-27

    🚫 被阻塞的计划:
      2. 项目A开发 (工作)
         该计划被阻塞,需要解决阻塞问题

    ⚠️ 即将到期的计划:
      3. 30天减肥计划 (运动)
         还有 2 天到期
         截止: 2026-04-01

    📅 长时间未更新的计划:
      4. 学习Python (学习)
         已 8 天未更新

    💡 建议:
      • 优先处理已逾期的计划,或考虑调整截止日期
      • 尽快解决阻塞问题,确保计划能够继续推进
      • 关注即将到期的计划,合理安排时间
      • 更新长时间未动的计划,保持进度记录
```

### 提醒类型说明

| 类型 | 判断规则 | 优先级 | 图标 |
|------|---------|--------|------|
| **overdue** | deadline < 今天 且 status ≠ completed | 🔴 最高 | 🔴 |
| **blocked** | status = "blocked" | 🚫 高 | 🚫 |
| **due_soon** | 今天 ≤ deadline ≤ 今天+3天 | ⚠️ 中 | ⚠️ |
| **stale** | updatedAt < 今天-7天 且 status = "in_progress" | 📅 低 | 📅 |

### 主动提醒场景

OpenClaw 可以在以下场景**主动**调用 `get_plan_reminders`:

1. **用户登录时** (每天首次)
   ```
   用户: "早上好"
   AI: 早上好!您今天有 2 项计划需要关注:
       • 准备年度报告(已逾期2天)
       • 30天减肥计划(还有2天到期)
   ```

2. **查询计划列表时**
   ```
   用户: "我有哪些计划?"
   AI: [显示计划列表]
       💡 提示: 您有 1 个已逾期的计划需要关注
   ```

3. **询问今日安排时**
   ```
   用户: "今天有什么安排?"
   AI: [先检查提醒,再显示今日计划]
   ```

```mermaid
用户说: "把减肥计划延长到4月1日"
  ↓

  ↓

返回成功消息
```

```mermaid
  ↓
返回分类的提醒列表
  ↓
AI格式化输出:
  🔴 已逾期
  🚫 被阻塞
  ⚠️ 即将到期
  📅 长时间未更新
```

## 📊 后端API

### 修改计划API

**请求**:
```http
POST /api/openclaw/plans/:planId/update
Authorization: Bearer <token>
Content-Type: application/json

{
  "deadline": "2026-04-01",
  "importance": 8
}
```

**响应**:
```json
{
  "success": true,
  "data": {
    "plan_id": "plan_abc",
    "updated_fields": ["deadline", "dueDate", "importance"],
    "change_summary": "deadline: 2026-03-29 → 2026-04-01; importance: 7 → 8",
    "message": "计划已更新: deadline, dueDate, importance"
  }
}
```

### 获取提醒API

**请求**:
```http
GET /api/openclaw/plans/reminders
Authorization: Bearer <token>
```

**响应**:
```json
{
  "success": true,
  "data": {
    "total_reminders": 4,
    "reminders": {
      "overdue": [...],
      "blocked": [...],
      "due_soon": [...],
      "stale": [...]
    },
    "summary": {
      "overdue_count": 1,
      "due_soon_count": 1,
      "stale_count": 1,
      "blocked_count": 1
    }
  }
}
```
