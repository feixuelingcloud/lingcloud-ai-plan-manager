# 测试创建计划功能

## 问题分析

根据用户反馈："创建计划遇到了 API 兼容性问题 — 后端接口需要额外参数（用户消息）"

### 后端接口要求

**接口**: `POST /api/openclaw/plans/draft`

**必填参数**:
- `goal`: 计划目标或标题（字符串）

**可选参数**:
- `constraints`: 约束条件或补充说明（字符串）
- `plan_type`: 计划类型（字符串）
  - work(工作)、study(学习)、life(生活)、exercise(运动)
  - hobby(兴趣)、travel(出行)、leisure(休闲)、other(其他)
- `granularity`: 时间颗粒度（字符串）
  - day(按天)、week(按周)、month(按月)，默认day
- `language`: 语言（字符串）
  - zh-CN(中文)、en(英文)，默认zh-CN

### 插件当前实现

插件已正确传递所有这些参数，代码位于 `src/tools/createPlanDraft.ts`。

### 3. 检查防火墙

在 Windows PC 上（10.111.222.1）：

```powershell

netsh advfirewall firewall show rule name="Node.js Server - Port 4000"

# 如果没有规则，添加防火墙规则
netsh advfirewall firewall add rule name="Node.js Server - Port 4000" dir=in action=allow protocol=TCP localport=4000
```

### 4. 检查后端监听地址

检查后端是否监听在正确的地址上：

```bash
# 在 Windows PC 上查看后端日志
# 应该看到类似：Server listening on http://0.0.0.0:4000
```

如果看到 `http://127.0.0.1:4000` 或 `http://localhost:4000`，则需要修改后端代码：

```javascript
// 错误（只能本机访问）
app.listen(4000, 'localhost', () => { ... });

// 正确（允许远程访问）
app.listen(4000, '0.0.0.0', () => { ... });
```

## 可能的错误原因

### A. 网络不通
- 防火墙阻止
- 后端只监听 localhost
- IP 地址错误

### B. 认证失败
- API Key 无效
- API Key 未正确配置到 OpenClaw

### C. 后端服务问题
- 后端未启动
- 千问 API 配置错误
- 数据库连接失败
