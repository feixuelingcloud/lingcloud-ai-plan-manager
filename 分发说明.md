## 👥 用户安装步骤 (简化版)

```bash
# 1. 安装插件
openclaw plugins install lingyun-openclaw-plan-manager-1.0.0.tgz

# 2. 配置
cd ~/.openclaw/plugins/@lingyun/openclaw-plan-manager
cp .env.example .env
# 编辑 .env 填入:
# AI_PLAN_API_BASE=http://your-server:4000/api
# AI_PLAN_API_TOKEN=your_token

# 3. 重启
openclaw gateway restart

# 4. 测试
# 在 OpenClaw 中说: "帮我制定一个计划"
```

## 🔧 用户需要知道的配置

### 1. 后端地址

用户需要知道后端服务的地址:
- 如果是本地部署: `http://localhost:4000/api`
- 如果是服务器部署: `http://服务器IP:4000/api` 或 `https://domain.com/api`

### 2. API Token

用户需要从系统获取Token:
1. 登录 AI计划管理系统
2. 进入"个人设置"
3. 生成 OpenClaw Token
4. 复制到配置文件

**注意**: 如果系统还没有Token生成功能,可以:
- 临时使用用户的 userId 作为 Token
- 或修改后端临时绕过认证(仅用于测试)

## 📋 常见问题

### Q1: macOS 用户无法执行 openclaw 命令?

**A**: 这是 macOS 的常见问题,请参考 [macOS安装指南.md](macOS安装指南.md),提供以下解决方案:
- 使用完整路径: `/Applications/OpenClaw.app/Contents/MacOS/openclaw`
- 添加到 PATH
- 通过图形界面安装
- 手动解压安装

### Q2: 用户说找不到插件配置文件?

**A**: 配置文件位置:
- Windows: `%USERPROFILE%\.openclaw\plugins\@lingyun\openclaw-plan-manager\.env`
- Linux/Mac: `~/.openclaw/plugins/@lingyun/openclaw-plan-manager/.env`

### Q3: 用户说连接失败?

**A**: 检查:
1. 后端服务是否运行?
2. 端口是否正确? (4000)
3. 防火墙是否允许?
4. Token是否有效?

### Q4: 用户想更新插件?

**A**:
```bash
# 卸载旧版本
openclaw plugins uninstall @lingyun/openclaw-plan-manager

# 安装新版本
openclaw plugins install lingyun-openclaw-plan-manager-1.1.0.tgz

# 重启
openclaw gateway restart
```
