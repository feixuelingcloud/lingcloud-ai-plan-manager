# OpenClaw 插件安装和测试指南

本指南将帮助您将 AI计划管理系统 插件安装到 OpenClaw 并进行测试。

---

## 📋 前置准备

### 1. 确认 OpenClaw 已安装

```bash
# 检查 OpenClaw 版本
openclaw --version

# 如果未安装,请访问 OpenClaw 官网安装
# https://openclaw.ai
```

### 2. 确认后端服务已启动

```bash
# 进入后端目录
cd backend

# 启动后端服务
npm start
# 或
node app.js

# 确认服务运行正常
# 应该看到类似输出: "Server running on http://localhost:3000"
```

### 3. 获取 API Token

**重要**: 您需要从 AI计划管理系统 获取 API Token

目前有两种方式:

**方式A: 如果系统已有 Token 生成功能**
1. 登录 AI计划管理系统 (http://localhost:3000)
2. 进入"个人设置" → "OpenClaw集成"
3. 点击"生成Token"
4. 复制生成的 Token

**方式B: 临时测试(如果还没有 Token 功能)**
使用用户ID作为临时 Token:
- 打开浏览器开发者工具
- 查看 localStorage 或 cookie 中的用户信息
- 使用 `userId` 或 `_openid` 作为临时 Token

## 📦 步骤1: 配置环境变量

```bash
# 进入插件目录
cd openclaw-plan-manager

# 复制环境变量示例文件
copy .env.example .env
# Linux/Mac 使用: cp .env.example .env
```

编辑 `.env` 文件,填入您的配置:

```env
# API 基础URL (根据您的后端端口调整)
AI_PLAN_API_BASE=http://localhost:3000/api

# API Token (从系统获取,或使用临时Token)
AI_PLAN_API_TOKEN=your_actual_token_here

# 请求超时时间(毫秒)
AI_PLAN_TIMEOUT_MS=15000

# 调试模式(可选)
DEBUG=false
```

**配置说明**:
- `AI_PLAN_API_BASE`: 您的后端 API 地址,注意要加 `/api` 路径
- `AI_PLAN_API_TOKEN`: 您的认证 Token,必须正确才能访问 API
- `AI_PLAN_TIMEOUT_MS`: API 请求超时时间,默认15秒

## 🔧 步骤2: 安装依赖

```bash
# 确保在 openclaw-plan-manager 目录下
npm install

# 如果遇到依赖问题,尝试:
npm install --legacy-peer-deps
```

**安装的主要依赖**:
- `@sinclair/typebox`: 参数类型验证
- `typescript`: TypeScript 支持

---

## 🚀 步骤3: 安装插件到 OpenClaw

### 方式A: 开发模式 (推荐用于测试)

开发模式会创建符号链接,代码修改后立即生效:

```bash
# 在 openclaw-plan-manager 目录下执行
openclaw plugins install -l .

# 或使用完整路径
openclaw plugins install --link "f:\菩提闻道\001-塞外飞雪2024\桌面\AI计划管理系统_网页版20260328\openclaw-plan-manager"
```

**优点**:
- 修改代码后立即生效,无需重新安装
- 方便开发和调试

### 方式B: 生产模式 (打包安装)

生产模式会打包插件为 `.tgz` 文件:

```bash
# 1. 打包插件
npm pack
# 会生成: lingyun-openclaw-plan-manager-1.0.0.tgz

# 2. 安装打包文件
openclaw plugins install lingyun-openclaw-plan-manager-1.0.0.tgz
```

**优点**:
- 独立部署
- 版本管理

---

## 🔄 步骤4: 重启 OpenClaw Gateway

安装插件后,需要重启 OpenClaw Gateway 才能加载插件:

```bash
# 重启 Gateway
openclaw gateway restart

# 或者先停止再启动
openclaw gateway stop
openclaw gateway start
```

等待几秒钟,让 Gateway 完全启动。

## 🔍 故障排查

### 问题1: 插件未出现在列表中

**可能原因**:
- 安装路径不正确
- package.json 配置有误
- OpenClaw Gateway 未重启

**解决方法**:
```bash
# 1. 卸载插件
openclaw plugins uninstall @lingyun/openclaw-plan-manager

# 2. 检查 package.json 中的 name 字段
# 应该是: "@lingyun/openclaw-plan-manager"

# 3. 重新安装
openclaw plugins install -l .

# 4. 重启 Gateway
openclaw gateway restart

# 5. 检查日志
openclaw gateway logs
```

---

### 问题2: 工具调用失败 "API连接错误"

**可能原因**:
- 后端服务未启动
- API_BASE URL 配置错误
- Token 无效

**解决方法**:

**步骤1: 检查后端服务**
```bash
# 确认后端正在运行
curl http://localhost:3000/api/health
# 或在浏览器访问
```

**步骤2: 检查环境变量**
```bash
# 查看插件配置
cat .env

# 确认:
# AI_PLAN_API_BASE=http://localhost:3000/api
# AI_PLAN_API_TOKEN=正确的token
```

**步骤3: 测试 API 连接**
```bash
# 使用 curl 测试 API
curl -H "Authorization: Bearer YOUR_TOKEN" http://localhost:3000/api/openclaw/plans
```

**步骤4: 查看详细错误**
```bash
# 开启调试模式
# 在 .env 中设置 DEBUG=true

# 重启插件
openclaw gateway restart

# 查看详细日志
openclaw gateway logs --follow
```

---

### 问题3: Token 认证失败

**临时解决方案**: 如果后端还没有实现 Token 生成功能,可以:

1. 修改后端中间件,临时关闭 Token 验证:

```javascript
// backend/src/middleware/openclawAuth.js
// 临时添加:
if (!token) {
  // 临时使用固定用户ID
  req.openclawUser = {
    userId: 'test_user_123',
    username: 'test_user'
  };
  return next();
}
```

2. 或者从数据库获取一个真实的用户ID,作为临时 Token 使用

# 检查 TypeScript 配置

npm install @types/node --save-dev

### 问题5: 草稿过期问题

**现象**: 提示 "草稿不存在或已过期"

**原因**: 草稿有效期为30分钟

**解决**: 重新生成草稿

---

### 问题6: 查询返回空列表

**可能原因**:
- 数据库中没有计划数据
- 时间范围筛选过滤了所有计划
- userId 不匹配

**解决方法**:

1. **检查数据库**:
```javascript
// 在数据库中查询
db.collection('tasks').where({
  _openid: 'your_user_id'
}).get()
```

2. **创建测试数据**:
```
先使用插件创建一个计划:
"帮我制定一个测试计划"
"确认"
```

3. **查看全部计划**:
```
"查看所有计划"  // 使用 time_range=all
```
