# ⚠️ 重要：必须重启后端服务

## 问题原因

之前插件调用 API 时使用的是 **PATCH** 方法，但后端路由只支持 **POST** 方法，导致请求失败或走到了错误的路由。

## 已修复的问题

### 1. 后端路由修复
文件：`AI计划管理系统_网页版/backend/src/routes/openclaw.js`

已添加 PATCH 方法支持：
```javascript
// 修改计划（支持 POST 和 PATCH 两种方式）
router.post('/plans/:planId/update', openclawController.updatePlan);
router.patch('/plans/:planId/update', openclawController.updatePlan);  // ← 新增

// 更新任务状态（支持 POST 和 PATCH 两种方式）
router.post('/tasks/:taskId/status', openclawController.updateTaskStatus);
router.patch('/tasks/:taskId/status', openclawController.updateTaskStatus);  // ← 新增
```

### 2. 插件修复
- ✅ 完全移除了 `status` 字段参数
- ✅ 只保留 `completed` 字段
- ✅ 使用 PATCH 方法调用 API

### 3. 后端同步逻辑修复
- ✅ 单向同步：`completed=true` → `status='completed'`
- ✅ 修改 `status` 不影响 `completed`

### 步骤 3：测试

测试标记计划完成功能：

1. 在 OpenClaw 中说："把第一个计划标记为完成"
2. 检查数据库：
   - `completed` 字段应该为 `true`
   - `status` 字段应该为 `'completed'`
   - `completedAt` 字段应该有时间戳

## 验证方法

### 检查后端日志
启动后端后，应该看到类似输出：
```
📝 更新计划 xxx, 字段: completed, status
```

### 检查数据库
MongoDB 中的 tasks 集合，计划文档应该包含：
```json
{
  "completed": true,
  "status": "completed",
  "completedAt": "2026-04-06T14:57:00.000Z"
}
```

## 如果还是不工作

1. **检查后端是否重启**
   - 查看后端日志，确认服务器已重新加载

2. **检查 API 请求**
   - 在后端日志中查看收到的请求方法（应该是 PATCH）
   - 查看请求体（应该包含 `{"completed": true}`）

3. **检查插件版本**
   - 确认安装的是最新版本（2026-04-06 22:57）
