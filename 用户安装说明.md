# AI计划管理系统 OpenClaw 插件 - 用户安装说明

本文档面向**最终用户**,说明如何在 OpenClaw 中安装和使用本插件。

---

## 📦 安装方式

### 方式1: 从 ClawHub 安装 (推荐)

如果插件已发布到 ClawHub:

1. 打开 OpenClaw 应用
2. 进入插件市场
3. 搜索 "AI计划管理" 或 "@lingyun/openclaw-plan-manager"
4. 点击安装
5. 配置插件(填入后端地址和Token)
6. 开始使用

---

### 方式2: 从安装包安装 (本地部署)

如果您获得了 `.tgz` 安装包文件:

#### Windows 用户:

```cmd
# 1. 打开命令提示符(cmd)
# 2. 进入安装包所在目录
cd C:\Downloads

# 3. 安装插件
openclaw plugins install lingyun-openclaw-plan-manager-1.0.0.tgz

# 4. 重启 OpenClaw Gateway
openclaw gateway restart
```

#### Linux/Mac 用户:

```bash
# 1. 打开终端
# 2. 进入安装包所在目录
cd ~/Downloads

# 3. 安装插件
openclaw plugins install lingyun-openclaw-plan-manager-1.0.0.tgz

# 4. 重启 OpenClaw Gateway
openclaw gateway restart
```

**macOS 用户注意**: 如果遇到 "openclaw: command not found" 错误,请查看 [macOS安装指南.md](macOS安装指南.md) 获取详细的 macOS 特定安装步骤。

---

## ⚙️ 配置插件

安装后需要配置后端连接:

### 配置方法1: 通过环境变量

在插件目录创建 `.env` 文件:

**Windows**:
```cmd
# 找到插件安装目录
cd %USERPROFILE%\.openclaw\plugins\@lingyun\openclaw-plan-manager

# 创建配置文件
echo AI_PLAN_API_BASE=http://your-server:4000/api > .env
echo AI_PLAN_API_TOKEN=your_token_here >> .env
```

**Linux/Mac**:
```bash
# 找到插件安装目录
cd ~/.openclaw/plugins/@lingyun/openclaw-plan-manager

# 创建配置文件
cat > .env << EOF
AI_PLAN_API_BASE=http://your-server:4000/api
AI_PLAN_API_TOKEN=your_token_here
AI_PLAN_TIMEOUT_MS=15000
EOF
```

### 配置方法2: 通过 OpenClaw 界面

如果 OpenClaw 支持插件配置界面:

1. 打开 OpenClaw
2. 进入"插件设置" → "AI计划管理系统"
3. 填入配置:
   - **API 地址**: `http://your-server:4000/api`
   - **API Token**: 从系统获取的Token
4. 保存配置

---

## 🔑 获取 API Token

联系系统管理员获取您的 API Token,或:

1. 登录 AI计划管理系统 (http://your-server:4000)
2. 进入"个人设置"
3. 找到"OpenClaw 集成"
4. 点击"生成Token"
5. 复制Token并粘贴到配置中

## 🔧 故障排查

### 问题1: 插件未出现在列表中

**解决**:
```bash
# 检查插件列表
openclaw plugins list

# 如果没有,重新安装
openclaw plugins install lingyun-openclaw-plan-manager-1.0.0.tgz

# 重启 Gateway
openclaw gateway restart
```

### 问题2: "API 连接失败"

**原因**:
- 后端服务未启动
- API 地址配置错误
- Token 无效

**解决**:
1. 检查后端服务是否运行
2. 确认 `.env` 中的 `AI_PLAN_API_BASE` 地址正确
3. 确认 Token 有效

### 问题3: "Token 认证失败"

**解决**:
1. 重新从系统获取 Token
2. 更新 `.env` 文件中的 `AI_PLAN_API_TOKEN`
3. 重启 OpenClaw Gateway

### 问题4: 功能不响应

**解决**:
```bash
# 查看日志
openclaw gateway logs

# 重启插件
openclaw gateway restart
```
