# 诊断：为什么还在修改 status 字段

## 当前情况

用户报告：通过 OpenClaw 更新计划状态时，API 还是只修改 `status` 字段，没有修改 `completed` 字段。

## 可能的原因

### 1. OpenClaw 使用的是旧版本插件

**如何验证**：
```bash
# 在 OpenClaw 中运行这个命令查看插件信息
# 或者检查 OpenClaw 的插件目录
```

**解决方法**：
1. 完全卸载旧插件
2. 清除 OpenClaw 缓存
3. 重启 OpenClaw
4. 重新安装新版本插件

### 2. 后端没有重启

**如何验证**：
检查后端启动时间和路由配置

**解决方法**：
```bash
cd AI计划管理系统_网页版/backend
# Ctrl+C 停止
npm start
```

### 3. 还有其他更新计划的路由或工具

**如何验证**：
检查是否有多个工具都在更新计划

## 立即诊断步骤

### 步骤 1：检查后端实际收到的请求

在后端 `openclawController.js` 的 `updatePlan` 函数开头添加日志：

```javascript
// 在 updatePlan 函数开头添加
console.log('🔍 收到更新计划请求:');
console.log('  - Plan ID:', planId);
console.log('  - Request body:', JSON.stringify(updateFields, null, 2));
console.log('  - HTTP Method:', req.method);
```

位置：`AI计划管理系统_网页版/backend/src/controllers/openclawController.js` 第 470 行附近

### 步骤 2：检查 OpenClaw 实际调用的工具

在 OpenClaw 中，当用户说"标记计划完成"时，查看 AI 的响应：
- AI 应该说："正在调用 update_plan 工具，参数: { plan_id: 'xxx', completed: true }"
- 如果 AI 说的是 `status: 'completed'`，说明使用的是旧版本插件

### 步骤 3：验证插件版本

检查 OpenClaw 中实际加载的插件代码：
1. 找到 OpenClaw 的插件安装目录
2. 查看 `@feixuelingcloud/lingcloud-ai-plan-manager` 文件夹
3. 检查 `dist/src/tools/updatePlan.js` 文件
4. 搜索 `status: Type.Optional`
   - **如果找到**：说明是旧版本
   - **如果没找到**：说明是新版本

## 强制清除旧版本的方法

### 方法 1：手动删除插件目录

1. 找到 OpenClaw 的插件目录（通常在用户目录下）
2. 删除 `lingcloud-ai-plan-manager` 相关的所有文件夹
3. 重启 OpenClaw
4. 重新安装

### 方法 2：使用不同的插件名称

临时解决方案：修改插件名称为新名称，强制 OpenClaw 识别为新插件

  "status": "completed"
}

**后端日志**：

  - Plan ID: abc123
